<h1 align="center">PackFoundry</h1>

<p align="center">
  Сборка Flutter-приложений под разные платформы и форматы в пару кликов.
</p>

PackFoundry — GUI-приложение на Flutter, которое помогает превратить Flutter-проект в готовые установщики и переносимые пакеты без ручной рутины в терминале.

Идея простая: выбрать папку проекта, указать название выходного пакета, иконку, папку экспорта, отметить нужные платформы и нажать Build. PackFoundry сам проверяет окружение, показывает доступные варианты сборки, запускает нужные инструменты и ведёт понятный пошаговый лог.

## Зачем это нужно

Flutter отлично собирает приложения, но релизная упаковка быстро превращается в набор повторяющихся действий: подготовить окружение, вспомнить команды, собрать bundle, настроить иконку, создать AppImage, `.deb`, `.rpm`, Windows installer, Android APK и так далее.

PackFoundry должен стать единым окном для этой работы:

- сборка Flutter-приложений под Linux, Windows, Android, macOS и iOS;
- упаковка в привычные форматы установщиков и переносимых пакетов;
- диагностика того, что уже установлено и чего не хватает;
- установка или подключение дополнительных инструментов сборки;
- изолированные окружения сборки там, где это нужно, например Docker для `.deb`;
- подробный визуальный прогресс вместо молчаливого терминала.

## Как это выглядит для пользователя

1. Открыть PackFoundry.
2. Выбрать папку Flutter-проекта.
3. Настроить иконку и параметры окна.
4. Выбрать платформы и типы пакетов.
5. Выбрать папку экспорта и имя выходного пакета.
6. Нажать Build.
7. Получить готовые артефакты в выбранной папке.

PackFoundry собирает проект во временной копии, поэтому исходники выбранного приложения не меняются.

## Текущий статус

Проект находится в активной разработке. Первый фокус — Linux-host workflow, но архитектура и интерфейс уже строятся вокруг будущей сборки под все основные платформы.

Уже реализовано:

- выбор Flutter-проекта и папки экспорта;
- выбор иконки `.png` или `.svg`;
- ввод имени выходного пакета;
- автоматическое чтение размера окна из Flutter desktop runner;
- применение размера окна во временной копии проекта без изменения исходников;
- рабочая сборка Linux AppImage;
- сборка Linux `.deb` внутри Docker-контейнера Debian, чтобы не зависеть от дистрибутива хоста;
- экспорт Linux `tar.gz` bundle;
- подробный лог и прогресс сборки;
- диагностика инструментов, включая Flutter SDK, Linux toolchain, Docker, Inno Setup и Android SDK;
- светлая, тёмная и системная тема;
- русская и английская локализация с автоопределением языка системы;
- приветственное окно с выбором темы.

В планах:

- полноценная сборка `.rpm`;
- Windows installer через Inno Setup;
- Android APK/AAB;
- macOS `.dmg` и iOS `.ipa` через нативный или удалённый builder;
- установка Flutter SDK, Docker, AppImageTool, Inno Setup и других зависимостей из интерфейса;
- более развитый интерактивный обзор возможностей и настройки.

## Рабочие пространства

### Settings

Настройки самой программы и диагностика инструментов сборки. Здесь видно, что уже установлено, а что нужно добавить, чтобы открыть больше целей сборки.

### Project

Выбор Flutter-проекта и параметров приложения:

- каталог проекта;
- ширина и высота окна;
- иконка `.png` или `.svg`;
- список пакетов, которые нужно собрать.

### Build

Финальный шаг перед сборкой:

- имя выходного пакета;
- папка экспорта;
- кнопка Build;
- подробный пошаговый лог выполнения.

| --- | --- | --- |
| Linux | AppImage | Реальная сборка |
| Linux | `.deb` | Реальная сборка через Docker/Debian |
| Linux | `tar.gz` | Реальная сборка bundle-архива |

## Как работает сборка

PackFoundry создаёт временную копию выбранного Flutter-проекта, применяет настройки только к этой копии, запускает сборку и упаковывает результат в выбранные форматы.

Для AppImage используется локальный Linux bundle и `appimagetool`. Если `appimagetool` не найден, PackFoundry пытается скачать его в кеш пользователя.

Для `.deb` используется Docker-контейнер Debian. Это позволяет собирать Debian-пакет не только на Debian/Ubuntu, но и на Fedora, Arch и других Linux-дистрибутивах, где доступен Docker.
